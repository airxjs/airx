## Code Architecture and Implementation Findings

### Nature of this Project

`airx` is a **JSX web application framework**, not a product with its own UI:
- Signal-based reactive state management
- Virtual DOM diffing and rendering
- Both browser and SSR rendering
- Plugin system for extensibility
- TypeScript-first

### No UI or Styling System

- **No styles.css** — no frontend UI
- **No web/ directory** — this is a library/framework
- **No design tokens** — delegated to users of the framework
- Framework intentionally defers styling to user choice (CSS, Tailwind, etc.)

### Framework's Relationship to UI

`airx` provides rendering primitives; styling is entirely the user's responsibility:
- `browserRender()` renders to DOM — no styling opinion
- `serverRender()` outputs HTML strings — no styling opinion
- No built-in CSS-in-JS, Tailwind, or any styling library
- Users choose their own design system
