## Code Architecture and Implementation Findings

### 当前状态 (Phase 1 完成后)

两份文件的 `commitDom` 函数结构如下：

```
commitDom(rootInstance, rootNode?)
  └─ updateDomProperties(dom, nextProps, prevProps)  [差异大]
  └─ commitInstanceDom(nextInstance, oldNode?)        [部分重复]
       ├─ deletions 处理                              [完全相同]
       ├─ dom 创建 (createElement/createTextNode)      [平台相关]
       ├─ updateDomProperties 调用                      [差异大]
       └─ 插入 parent (insertBefore/appendChild)       [完全相同]
  └─ createCommitWalker({ commitInstanceDom, ... })   [已共享 Phase 1]
```

### 完全相同的代码段

#### 1. Deletion 处理逻辑 (browser.ts#L97-106, server.ts#L289-298)

```typescript
// browser 和 server 完全相同
if (nextInstance.deletions) {
  for (const deletion of nextInstance.deletions) {
    const domList = getChildDoms(deletion)
    for (let index = 0; index < domList.length; index++) {
      const dom = domList[index]
      if (dom && dom.parentNode) dom.parentNode.removeChild(dom)
    }
    deletion.context.triggerUnmounted()
  }
  nextInstance.deletions.clear()
}
```

#### 2. Parent 插入逻辑 (browser.ts#L149-165, server.ts#L327-343)

```typescript
// browser 和 server 完全相同
if (nextInstance.domRef != null) {
  if (oldNode !== nextInstance.domRef) {
    const parentDom = getParentDom(nextInstance)
    if (nextInstance.domRef.parentNode === parentDom) {
      parentDom.insertBefore(nextInstance.domRef, oldNode ?? null)
    } else {
      if (nextInstance.domRef.parentNode) {
        nextInstance.domRef.parentNode.removeChild(nextInstance.domRef)
      }
      parentDom.appendChild(nextInstance.domRef)
    }
  }
}
```

#### 3. getDebugElementName 辅助函数 (browser.ts#L84-93, server.ts#L276-285)

完全相同的辅助函数，在两份文件中重复定义。

### 差异较大的代码段

#### updateDomProperties

- **Browser**: 调用 `plugin.updateDom(dom, nextProps, prevProps)`，将属性更新委托给插件系统
- **Server**: 实现完整的 props diff 逻辑，包含 isKey/isRef/isStyle/isClass/isEvent/isProperty 等判断

Server 的 `updateDomProperties` 中的 props 分类逻辑可以提取为共享的 `prop-utils.ts`，但 DOM 操作（setAttribute/removeAttribute）保持平台特定。

#### DOM 创建

- **Browser**: `document.createElementNS`, `document.createTextNode`, `document.createComment`
- **Server**: `ServerElement.createElement`, `ServerElement.createTextNode`, `ServerElement.createComment`

这部分是平台相关的，不能提取。

## Implementation Plan

### 步骤 1: 创建 `source/render/basic/commit-helpers.ts`

提取以下共享内容：

1. **`getDebugElementName`** - 两份完全相同的实例名获取函数
2. **`removeDeletions`** - 两份完全相同的删除处理逻辑
3. **`insertDomIntoParent`** - 两份完全相同的 parent 插入逻辑
4. **`createPropClassifier`** - Server updateDomProperties 中的 props 分类逻辑（isKey, isRef, isStyle, isClass, isEvent, isChildren, isProperty, isGone, isNew）
