## Code Architecture and Implementation Findings

### 0.7.x 现状总结

**已完成**:
- ✅ SSR 字符串渲染 (`renderToString`)
- ✅ 基础 SSR API (`createSSRApp`, `SSRApp`)
- ✅ SSR 测试门禁与文档示例
- ✅ Browser 调度器优化（无工作时不空转）
- ✅ `requestIdleCallback` 降级处理
- ✅ 17 个 ROADMAP-0003 SSR 任务全部 DONE

**未完成 / 待实现**:
- ❌ `hydrate()` - 当前为 stub（no-op），README 标注为 0.8.x 计划
- ❌ `AirxApp.plugin()` - 标记 @deprecated WIP
- ❌ `AirxApp.renderToHTML()` - 标记 @deprecated WIP

**技术债**:
- Browser / Server 的 `commitDom` 逻辑有较高重复度
- Signal watcher 的微任务收集策略仍需更多集成测试验证
- SSR 与 Browser 的 DOM 属性更新策略尚未完全统一
- README / API 文档与代码实现存在不一致

## Risks and Open Questions

1. **Hydrate 实现复杂度** - 客户端 hydration 需要 DOM diff 和事件绑定，复杂度较高
2. **`plugin()` / `renderToHTML()` 去留** - 需要在 0.8.x 中确定
3. **Browser/Server 提交逻辑收敛** - 可能影响 SSR 输出，需要回归测试
4. **下游包版本对齐** - router、vite-plugin 需要同步更新
