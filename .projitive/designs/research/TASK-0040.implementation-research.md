## Code Architecture and Implementation Findings

### browser.ts (L134-L160)
```typescript
// 插入 parent
// TODO: 针对仅移动时优化
if (nextInstance.domRef != null) {
  if (oldNode !== nextInstance.domRef) {
    if (nextInstance.domRef.parentNode) {
      nextInstance.domRef.parentNode.removeChild(nextInstance.domRef)
    }
    const parentDom = getParentDom(nextInstance)
    // ... error check ...
    parentDom.appendChild(nextInstance.domRef)
  }
}
```

**当前行为**：
- 当 `oldNode !== nextInstance.domRef` 时，说明 DOM 节点位置不对
- 先执行 `removeChild` 从当前父节点移除
- 再执行 `appendChild` 添加到新父节点

**优化机会**：
- 使用 `insertBefore` 直接移动，而不是 remove + append
- `insertBefore` 可以在同一父节点内高效移动元素
- 如果 `nextInstance.domRef.parentNode === parentDom`，直接 `parentDom.insertBefore(nextInstance.domRef, oldNode)` 即可

```typescript
// 优化后
if (nextInstance.domRef != null) {
  if (oldNode !== nextInstance.domRef) {
    const parentDom = getParentDom(nextInstance)
    if (
      parentDom.nodeType === Node.TEXT_NODE
      || parentDom.nodeType === Node.COMMENT_NODE
    ) {
      const parentInstance = (parentDom as { airxInstance?: Instance<BrowserElement> }).airxInstance
      throw new Error(...)
    }
    // 关键优化：如果已经是正确父节点，使用 insertBefore 移动
    // 否则 remove + append（不同父节点间移动）
    if (nextInstance.domRef.parentNode === parentDom) {
      // 同父节点内移动
      parentDom.insertBefore(nextInstance.domRef, oldNode)
    } else {
      // 跨父节点移动
      if (nextInstance.domRef.parentNode) {
        nextInstance.domRef.parentNode.removeChild(nextInstance.domRef)
      }
      parentDom.appendChild(nextInstance.domRef)
    }
  }
}
```

## Risks and Open Questions

- **风险**: 需要确保 `insertBefore` 的第二个参数 `oldNode` 在 DOM 中仍然存在（如果 `oldNode` 已被移动到别处，可能需要调整）
- **假设**: `oldNode` 参数在调用时仍然指向 DOM 中的有效节点
- **边界情况**: 
  - `oldNode` 为 null 时，`insertBefore(node, null)` 等同于 `appendChild`
  - 元素已经在正确位置时，`insertBefore` 是空操作
