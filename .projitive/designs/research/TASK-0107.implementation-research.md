## 2. Code Architecture Findings

### Current Public API Surface (source/index.ts)
- `createApp(element)` → AirxApp
- `createElement(type, props, ...children)` → AirxElement
- `hydrate(app, container, options?)` → Promise<void> (stable since 0.8)
- `renderToString(element)` → string (SSR)
- `Signal.State`, `Signal.Computed`, `Signal.Effect`, `Signal.Watcher`

### Subpath Exports (0.9.0)
- `./` — main entry
- `./jsx-runtime` — automatic JSX runtime
- `./jsx-dev-runtime` — development JSX runtime
- `./browser` — browser-only rendering (tree-shakeable for SSR)
- `./server` — server-only rendering

### Plugin System
- `plugin.install(app)` hook for extensibility
- `plugin.updateDom` boundary documented

### Experimental Markers
- No remaining `@experimental` markers in public API (removed in 0.8.0 for hydrate)
- Only 2 `@deprecated` in ARIA types (types.ts#L166, L183) — not functional API
