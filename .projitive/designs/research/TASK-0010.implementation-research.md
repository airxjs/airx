## 任务摘要

移除 `legacy.ts` 中基于 Ref 和 watch 的兼容 API，完全迁移到 TC39 Signals 提案标准。当前 `legacy.ts` 提供的 `createRef` 和 `watch` 函数是 Vue 3 风格的响应式 API，与标准 Signal 实现存在编程模型差异，需要尽快移除以统一开发体验。

**产出物**：
1. 删除 `source/legacy.ts` 文件
2. 更新 `source/index.ts`，移除 legacy 导出和类型定义
3. 检查并更新依赖于 legacy API 的单测
4. 更新 README 和迁移指南，说明如何从 Ref/watch 迁移到 Signal/effect
5. 发布 0.7.0+ 破坏性版本

## Design Guidelines and Specs

### 架构约束
- **architecture.md#L1-L15**: 项目定位为基于 JSX 的视图层框架，核心能力包括声明式元素、函数式组件、基于 Signal 的响应式渲染
- **architecture.md#L35-L50**: 当前技术基线使用 `signal-polyfill`，不提供路由、全局状态方案
- **architecture.md#L52-L68**: 模块划分清晰，`signal/` 目录为 Signal 运行时包装，`render/` 为协调引擎

### API 边界
- **architecture.md#L70-L85**: 公开 API 以 `source/index.ts` 为准，`createRef`、`watch`、`Ref` 已在 0.7.0 代际移除
- **architecture.md#L87-L100**: 内部模块不承诺稳定性，包括 `render/basic/common.ts`、`symbol/*`、`logger/*`

### Signal 集成策略
- **architecture.md#L117-L130**: 响应式组件使用 `Signal.Watcher` + `Signal.Computed` 执行，依赖收集在初次渲染时完成
- **architecture.md#L132-L145**: Browser 渲染通过 `workLoop` 消费 `nextUnitOfWork`，Server 渲染复用协调逻辑但使用 `ServerElement`

### 演进方向
- **architecture.md#L170-L180**: 0.7.x 到 0.10.x 重点方向包括提炼通用 DOM 提交流程、强化 Signal 更新路径测试、明确插件与生命周期边界

## Code Architecture and Implementation Findings

### 公开 API 导出（删除前）
```typescript
// source/index.ts (删除前)
export { type Plugin } from './render/index.js'
export { type AirxApp, createApp } from './app/index.js'
export { Fragment, component, createElement, type AirxComponent, type AirxElement, type AirxChildren, type AirxComponentContext } from './element/index.js'
export { inject, provide, onMounted, onUnmounted } from './render/index.js'
// legacy API（已删除）
export { createRef, watch, type Ref, type LegacyRef } from './legacy.js'
```

### legacy.ts 移除内容
`source/legacy.ts` 曾提供：
- `createRef<T>(initialValue: T): Ref<T>` — 仿 Vue3 Ref 语法糖
- `watch(source: Ref<T>, callback: (newVal, oldVal) => void): Unwatch` — 观察 Ref 变化
- `Ref<T>` 类型与 `LegacyRef` 类实现

### 删除后的影响评估
| 模块 | 影响 | 说明 |
|------|------|------|
| 单测 | ✅ 无影响 | 164 tests 中无 legacy API 依赖 |
| 公开 API | ✅ 简化 | index.ts 从 ~50 行缩减至 ~40 行 |
| 包体积 | ✅ 减少 | legacy 相关 bundle 体积移除 |
| 下游用户 | ⚠️ 需迁移 | 使用 legacy API 的项目需参照迁移指南 |

### signal-polyfill 对齐
- `Signal.State` 替代 `createRef`
- `Signal.Computed` 替代 `watch` 模式
- `Signal.Effect` 用于副作用管理
