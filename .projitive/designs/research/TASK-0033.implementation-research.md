### 2.3 Key Implementation Details

#### 核心接口
```typescript
export interface HydrateOptions {
  stateSnapshot?: StateSnapshot  // Signal 状态快照
  forceReset?: boolean           // 跳过 SSR 状态恢复
}

export interface HydratedApp {
  container: HTMLElement
  unmount: () => void
}

export function hydrate(
  element: AirxElement,
  container: HTMLElement,
  options?: HydrateOptions
): HydratedApp
```

#### 实现流程
1. **恢复 Signal 状态**（如有 stateSnapshot 且非 forceReset）
2. **构建 Instance 树** - 通过 performUnitOfWork
3. **连接 DOM** - 将 instance 树匹配到现有 DOM
4. **设置响应式更新** - 启用 reactive work loop

## 4. Architecture Notes

### 4.1 Hydrate vs Browser Render

| 特性 | Browser Render | Hydrate |
|------|----------------|---------|
| DOM 创建 | 是 | 否（复用现有） |
| Instance 树构建 | 是 | 是 |
| Reactive updates | 是 | 是 |
| Event listeners | 自动绑定 | 需要匹配现有 |

### 4.2 Signal State Challenge

Signal 状态的序列化需要：
1. SSR 时追踪被读取的 Signal
2. 序列化 Signal 的当前值
3. Hydrate 时反序列化并恢复

这需要修改 SSR 渲染流程，目前作为占位符实现。
