## Code Architecture and Implementation Findings

### 问题分析

**34 个 lint 错误，分为两类：**

1. **`@typescript-eslint/no-explicit-any` (29 处)**
   - 测试代码中使用 `Record<string, any>` 作为 props 类型
   - 使用 `as any` 强转为 `Instance` 类型
   - 原因：测试代码需要模拟 `Instance` 对象但未使用正确的类型

2. **`@typescript-eslint/no-unused-vars` (5 处)**
   - `nextInstance` 变量赋值后未使用
   - `ListItem` 函数定义后未使用
   - `parentContext` 变量赋值后未使用
   - `newElements` 变量赋值后未使用
   - `.map((item, i) =>` 中的 `i` 参数未使用

### 修复方案

1. **类型修复**
   - 导入 `Instance` 类型：`import { Instance } from './common.js'`
   - `createMockElement` 函数：`props: Record<string, any>` → `props: Record<string, unknown>`
   - 所有 `as any` → `as Instance`

2. **未使用变量修复**
   - 删除未使用的 `ListItem`、`parentContext`、`newElements` 定义
   - `.map((item, i) =>` → `.map(item =>` 移除未使用索引
   - 保留 `nextInstance` 赋值（测试 actual 使用了 `expect(nextInstance).toBe(...)`）
