## 2. Code Architecture Findings

### Current Public API Surface (source/index.ts)
```typescript
// Types
export * from './types/index.js'        // JSX/DOM attribute types

// App
export { type AirxApp, createApp } from './app/index.js'

// Element
export { Fragment, component, createElement,
         type AirxComponent, type AirxElement,
         type AirxChildren, type AirxComponentContext } from './element/index.js'

// Error Boundary
export { ErrorBoundary, createErrorBoundary,
         type ErrorBoundaryProps, type ErrorBoundaryState, type ErrorBoundaryRef } from './element/index.js'

// Hooks
export { inject, provide, onMounted, onUnmounted } from './render/index.js'

// Plugin
export { type Plugin } from './render/index.js'
```

### Subpath Exports (package.json)
- `./` — main entry (airx)
- `./jsx-runtime` — automatic JSX runtime
- `./jsx-dev-runtime` — development JSX runtime
- `./browser` — browser rendering + hydrate
- `./server` — SSR rendering (renderToString, createSSRApp)

### SSR API (source/render/server/server.ts)
```typescript
export interface SSRApp {
  renderToString(): Promise<string>
  hydrate(container: HTMLElement, options?: HydrateOptions): void
}
export function createSSRApp(element: AirxElement | AirxComponent): SSRApp
export function renderToString(app: SSRApp): Promise<string>
export function hydrate(_html: string, container: HTMLElement, app: SSRApp, options?: HydrateOptions): void
export type { HydrateOptions } from '../browser/index.js'
```

### Signal Primitives
- TC39 Signal standard: `Signal.State`, `Signal.Computed`, `Signal.Effect`, `Signal.Watcher`
- NOT re-exported from airx main entry (used directly from 'signal-polyfill')

## 3. Open Issues from Prior Audit

### Issue 3: element/index.ts exports non-public APIs
- `Props`, `AirxComponentRender`, `ReactiveComponent`, `isValidElement`, etc.
- Not re-exported from main entry, but exist in module
- **Risk**: Medium — users importing directly from `airx/element` may use unstable APIs
- **Recommendation**: Add `/** @internal */` JSDoc tags

### Issue 4: Internal module export chain
- `render/basic/common.ts` exports internal types that flow through export chain
- `INTERNAL_TEXT_NODE_TYPE`, `INTERNAL_COMMENT_NODE_TYPE`, `Disposer`, `InnerAirxComponentContext`, etc.
- **Risk**: Low — architecture doc says these are internal, rarely used directly
- **Recommendation**: Consider `stripInternal` TypeScript option
