# TASK-0034 研究简报：browser/server 提交流程重复分析

## 1. 背景

基于 `optimization-roadmap-0.8.md` Iteration B 阶段目标：**提取 browser/server 共用的 commit helper**，减少重复实现。

当前问题：`source/render/browser/browser.ts` 和 `source/render/server/server.ts` 中存在大量结构相同的 DOM 操作函数，导致：
- 维护成本高：修改一处逻辑需要同步修改另一处
- 回归风险：两端行为不一致
- 可测试性差：重复测试逻辑

### 2.3 `getParentDom` — 父节点查找

| 位置 | 行号 | 实现 |
|------|------|------|
| `browser/browser.ts` | 79 | 递归向上查找有 `domRef` 的 ancestor |
| `server/server.ts` | 227 | 同上 |

**重复程度：100%**（实现完全一致，仅类型不同）

```typescript
function getParentDom(instance: Instance<Element>): Element {
  if (instance.parent?.domRef != null) {
    return instance.parent.domRef
  }
  if (instance.parent) {
    return getParentDom(instance.parent)
  }
  throw new Error('Cant find dom')
}
```

### 2.4 `getChildDoms` — 子节点批量查找

| 位置 | 行号 | 实现 |
|------|------|------|
| `browser/browser.ts` | 95 | BFS 遍历所有子代实例，返回所有有 `domRef` 的节点 |
| `server/server.ts` | 243 | 同上 |

**重复程度：100%**（实现完全一致）

```typescript
function getChildDoms(instance: Instance<Element>): Element[] {
  const domList: Element[] = []
  const todoList: Instance<Element>[] = [instance]
  while (todoList.length > 0) {
    const current = todoList.pop()
    if (current?.domRef != null) {
      domList.push(current.domRef)
    }
    if (current?.domRef == null && current?.child != null) {
      todoList.push(current.child)
    }
    if (current?.sibling != null) {
      todoList.push(current.sibling)
    }
  }
  return domList
}
```

### 2.5 `commitInstanceDom` — 单实例 DOM 提交

| 位置 | 行号 | 职责 |
|------|------|------|
| `browser/browser.ts` | 123 | 单实例 DOM 创建/更新/删除 |
| `server/server.ts` | 271 | 同上 |

**重复程度：高**（结构一致，细节差异）

两端都处理：
- 函数组件调用（`typeof instance.element.type === 'function'`）
- 新增节点插入（`insertBefore` / `appendChild`）
- 删除节点处理
- 文本节点更新

差异点：
- `browser.ts` 有 `insertBefore`/`appendChild` 等 DOM API
- `server.ts` 有父子节点关系重建逻辑

## 3. 重构方案建议

### 方案 A（推荐）：共享 common 模块 + 平台特定 adapter

```
source/render/basic/common.ts     ← 已存在，可扩展
source/render/shared/commit/      ← 新增共用 commit helpers
  ├── getParentDom.ts
  ├── getChildDoms.ts
  ├── updateDomProperties.ts
  └── types.ts
```

**优点**：
- `getParentDom`、`getChildDoms` 逻辑完全一致，可直接共享
- `updateDomProperties` 可共享，仅类型泛型化
- `commitInstanceDom` 可保留平台差异，通过 adapter 注入 DOM 操作

**风险**：泛型化可能影响 TypeScript 类型推导，需验证

### 方案 B：将重复函数提取到 `common.ts`

直接复用现有的 `source/render/basic/common.ts`，将 `getParentDom`、`getChildDoms` 等泛型化后移入。

**优点**：无需新建目录
**缺点**：可能让 `common.ts` 职责膨胀
