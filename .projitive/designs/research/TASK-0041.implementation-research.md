## Code Architecture and Implementation Findings

### 问题 1: browser.ts 类型错误

**文件**: source/render/browser/browser.ts#L160  
**错误**: `Argument of type 'ChildNode | undefined' is not assignable to parameter of type 'Node | null'`

```typescript
// 修复前（编译错误）
parentDom.insertBefore(nextInstance.domRef, oldNode)

// 修复后
parentDom.insertBefore(nextInstance.domRef, oldNode ?? null)
```

**原因**: `oldNode` 参数类型为 `ChildNode | undefined`，但 `insertBefore` 的第二个参数要求 `Node | null`。`undefined` 不能直接赋值给 `Node | null`，需要用 `?? null` 转换。

### 问题 2: server.ts 缺少 insertBefore 方法

**文件**: source/render/server/server.ts#L76-113  
**错误**: `Property 'insertBefore' does not exist on type 'ServerElement'`

`ServerElement` 类只有 `removeChild` 和 `appendChild`，没有 `insertBefore` 方法。TASK-0040 的优化代码在 server.ts 中调用了 `parentDom.insertBefore(...)` 但 `ServerElement` 不支持此方法。

**修复方案**: 为 `ServerElement` 类添加 `insertBefore` 方法：

```typescript
insertBefore(dom: ServerElement, refNode: ServerElement | null) {
  if (refNode === null) {
    return this.appendChild(dom)
  }
  const refIndex = this.children.findIndex(v => v === refNode)
  if (refIndex === -1) {
    return this.appendChild(dom)
  }
  // @ts-ignore
  dom.parentNode = this
  // Update nextSibling of previous sibling
  if (refIndex > 0) {
    // @ts-ignore
    this.children[refIndex - 1].nextSibling = dom
  } else {
    // @ts-ignore
    this.firstChild = dom
  }
  // @ts-ignore
  dom.nextSibling = refNode
  this.children.splice(refIndex, 0, dom)
}
```

**同时修复**: server.ts 中两处调用 `insertBefore` 的地方使用 `oldNode ?? null`
