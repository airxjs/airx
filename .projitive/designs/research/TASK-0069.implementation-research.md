## Code Architecture and Implementation Findings

### 1. New `ssr-state.ts` Module
**Location**: `source/render/ssr/ssr-state.ts`

Core APIs:
- `registerSSRSignal(id, signal)` — L46-55: Registers Signal.State instances during SSR for tracking
- `getRegisteredSignals()` — L58-66: Returns all signals as serializable map
- `generateStateSnapshot()` — L69-76: Creates `StateSnapshot` { signals, version, timestamp }
- `clearSSRSignals()` — L82-84: Clears registry between SSR renders
- `injectStateSnapshotIntoHTML(html, snapshot)` — L90-107: Appends `<script type="airx/ssr-state">` to HTML

Key insight: This enables stateful SSR — signal values are captured during render and embedded in the HTML for client-side restoration.

### 2. Updated `server.ts` (SSR Renderer)
**Location**: `source/render/server/server.ts#L314`

- Line 314: `clearSSRSignals()` called in `serverRender()` before each render to avoid stale state
- This confirms SSR renderer now integrates with the SSR state module

### 3. Hydration Flow (Client-side)
**Location**: `source/render/browser/hydrate.ts`

- `hydrate(rootElement, container, options)` — Activates SSR HTML with event listeners + reactive state
- State restoration from the `airx/ssr-state` script tag injected during SSR

### 4. Version Reference
- Current version: `0.7.10` (from `package.json`)
- `ssrVersion` is hardcoded in `ssr-state.ts` but exposed via `setSSRVersion()`

## Risks and Open Questions

1. **Hydration state snapshot format** — The `StateSnapshot` interface lives in `browser/hydrate.ts`. Architecture doc should reference it but not duplicate type definitions.
2. **`setSSRVersion()`** — Currently only called internally but could be used by external callers. Architecture doc should note version handling.
3. **Version drift** — `ssrVersion` is hardcoded to `'0.7.10'`. If not updated before release, could cause snapshot compatibility issues. Not a documentation issue but worth noting.
4. **StateSnapshot serialization** — The JSON serialization in `injectStateSnapshotIntoHTML` handles empty signals gracefully (returns original HTML). This is a good edge case handling pattern.
