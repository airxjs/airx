## Code Architecture and Implementation Findings

### Browser Scheduler (source/render/browser/browser.ts)

**scheduleWorkLoop (lines ~254-261):**
```typescript
function scheduleWorkLoop() {
  if (context.isWorkLoopScheduled || context.nextUnitOfWork == null) {
    return
  }
  context.isWorkLoopScheduled = true
  scheduleIdleWork(deadline => {
    context.isWorkLoopScheduled = false
    workLoop(deadline)
  })
}
```
Key guards: double-scheduling prevention (`isWorkLoopScheduled`) and no-work prevention (`nextUnitOfWork == null`).

**scheduleIdleWork (lines ~28-35):**
```typescript
function scheduleIdleWork(callback: (deadline: IdleDeadline) => void) {
  if (typeof globalThis.requestIdleCallback === 'function') {
    globalThis.requestIdleCallback(callback)
    return
  }
  globalThis.setTimeout(() => callback(createIdleDeadline()), 0)
}
```
Fallback path: uses `setTimeout` when `requestIdleCallback` is unavailable. The `typeof` check is safe even when property is `undefined`.

**workLoop (lines ~129-162):**
```typescript
function workLoop(deadline: IdleDeadline = createIdleDeadline()) {
  while (context.nextUnitOfWork && !shouldYield) {
    // ...
    if (deadline) shouldYield = deadline.timeRemaining() < 1
  }
  // ...
  if (context.nextUnitOfWork != null) {
    scheduleWorkLoop()
  }
}
```
Key: if `nextUnitOfWork` is null after processing, `scheduleWorkLoop` is NOT called (no idle scheduling after sync completion).

**createIdleDeadline (lines ~21-25):**
```typescript
function createIdleDeadline(): IdleDeadline {
  return {
    didTimeout: false,
    timeRemaining: () => Number.MAX_SAFE_INTEGER
  }
}
```
`timeRemaining()` returns `MAX_SAFE_INTEGER` so `shouldYield` is always false during sync render — all sync work completes in one go.
